## 1. 分词器基础

### 1.1 分词器是什么？

**分词器 = 人类文字 → 机器数字的翻译器**

```
人类语言: "I believe that courage is the first step."
    ↓  分词器 (Tokenizer)
机器数字: [2, 45, 128, 39, 17, 612, 25, 7, 58, 401, 12, 3]
```

模型不认字母、不认单词，只认数字。分词器的工作：

- **编码 (encode)**: 文本 → token ID 列表
- **解码 (decode)**: token ID 列表 → 文本

### 1.2 完整流程

```
文本 "The brave knight"
  → encode →  [BOS, 234, 89, 512]
  → 模型推理
  → 输出 [BOS, 234, 89, 512, 78, 301, 45]
  → decode → "The brave knight drew his sword"
```

### 1.3 简单分词器 vs BPE 分词器

| | SimpleTokenizer (字符级) | BPE Tokenizer (字节对编码) |
|---|---|---|
| **原理** | 每个字符一个 ID | 按频率合并子词 |
| **"courage"** | 7 个 token: `c o u r a g e` | 1 个 token: `[1203]` |
| **"unbelievable"** | 12 个 token | 3 个 token: `un + believ + able` |
| **优点** | 极简，零依赖 | 高频词完整保留，低频词合理拆分 |
| **缺点** | 序列长，语义丢失 | 需要先训练分词器 |

```python
# SimpleTokenizer 的核心算法
for ch in text:
    tid = 4 + (ord(ch) % (vocab_size - 4))  # Unicode 码点取模映射
    ids.append(min(tid, vocab_size - 1))
```

## 2. 自定义 vs 开源

### 2.1 对比

| | 自定义 8K BPE | 开源通用 (GPT-2/LLaMA) |
|---|---|---|
| **词表大小** | 8,000 | 50,000 ~ 100,000+ |
| **训练语料** | TinyStories + 勇气材料 | 海量通用网页 |
| **"courage"** | ✅ 大概率独立 token | ❌ 大概率被拆分 |
| **embedding 参数量** | 8000 × 384 = **3M** | 50000 × 384 = **19.2M** |
| **总模型大小** | ~16M (25M budget) | ~32M+ |

### 2.2 为什么要自定义？

**1. 参数量控制**

```
embedding 矩阵 = vocab_size × n_embd
  8,000 × 384 =  3,072,000 参数  ← 只占 25M 预算的 12%
 50,000 × 384 = 19,200,000 参数  ← 占 25M 预算的 77%，几乎没空间给"思考层"
```

**2. 领域适配**

```
通用分词器: "SkullGreymon" → "Skull" + "Gre" + "ymon"  → 3 tokens
自定义分词器: "SkullGreymon" → [1203]                   → 1 token
```

**3. 简单直接** — 小词表 = 更快训练、更少内存

> 💡 **比喻**: 自定义字典就是只收录你需要的词，又小又准。  
> 用别人的大字典就像给摩托车装卡车油箱——重得跑不动。

## 3. 数值预测场景

### 3.1 与语言模型的根本区别

| | 语言模型 | 生产线质量预测 |
|---|---|---|
| **输入** | 文字 "I believe..." | 66 个浮点数 (50温度 + 10压力 + 1流速 + 5熔点) |
| **输出** | 下一个 token (8000选1) | 一个浮点数 (边压值) |
| **任务类型** | 分类 | **回归** |
| **需要分词器** | ✅ 是 | ❌ **不需要** |
| **损失函数** | CrossEntropyLoss | **MSE Loss** |
| **输入层** | Embedding(vocab, dim) | Linear(66, dim) |
| **输出层** | Linear(dim, vocab) | Linear(dim, 1) |

### 3.2 为什么不需要分词器？

分词器把**离散文字符号** → 整数。  
温度、压力、流速**本来就是数字**，直接喂：

```
文字流程: "courage" → tokenizer → [1203] → Embedding[1203] → 384维向量
数值流程: [195.2, 198.7, ..., 1.05, ...] → 归一化 → 66维向量 → Linear → 384维向量
```

### 3.3 时序维度设计

```
生产线的当前质量 ← 取决于 ← 过去一段时间的状态

输入: (batch=32, seq_len=20, features=66)
                          ↑ 过去 20 个时间步
输出: (batch=32, 1)  ← 预测当前时刻的边压
```

GSA 优势：注意力机制自动学习"过去哪个时刻的温度波动最关键"。

### 3.4 完整传感器参数

| 参数 | 数量 | 典型范围 | 说明 |
|---|---|---|---|
| 温度点 | 50 | 190~230°C | 沿挤出机各段 |
| 压力点 | 10 | 0.8~1.2 MPa | 熔体压力 |
| 流速 | 1 | 0.5~4 | 挤出速度 |
| 材料熔点 | 5 | 各材料不同 | PP约160°C |
| **合计输入** | **66** | | |

**建议补充**（如有传感器）：螺杆转速、牵引速度、冷却辊温度、原料含水率、机头压力、环境温湿度、回料比例、运行时长。

### 3.5 模型改动量

```python
# 改前（语言模型）      改后（回归模型）
self.emb = Embedding()   →  self.input_proj = Linear(66, 384)   # 输入层
self.head = Linear(8000) →  self.head = Linear(384, 1)         # 输出层
CrossEntropyLoss         →  MSE_Loss                            # 损失函数
# GSA Block / LayerNorm / 注意力 → 完全不动
```

## 4. 模型如何学习

### 4.1 核心四步

```
学习 = 预测 → 看差多少 → 算每个参数改多少 → 改一点 → 重复
    = forward →  loss   →    backward     →  step  → loop
```

### 4.2 一步步拆解

```
初始状态: 模型有 1600 万个权重（随机初始化）

第 1 步：预测 (Forward)
  输入: [195.3, 198.1, ..., 0.95, ..., 2.1, ...]  ← 66 个传感器
  经过 8 层 GSA 注意力 + FFN，输出: 3.1
  
第 2 步：看差多少 (Loss)
  真实边压 = 4.2，预测 = 3.1
  Loss = (3.1 - 4.2)² = 1.21

第 3 步：反向传播 (Backward)
  把 1.21 的误差沿网络链往回传
  计算每个权重对错误的"贡献"：∂loss/∂W
  
  例如:
    ∂loss/∂W₇   = +0.03   ← W₇ 调大会让误差更大 → 应该调小
    ∂loss/∂W₂₃  = -0.15   ← W₂₃ 调大会让误差变小 → 应该调大（没用够）
    ∂loss/∂W₅₁₂ = 0.0001  ← 几乎不影响 → 不动

第 4 步：更新权重 (Step)
  W_new = W_old - learning_rate × ∂loss/∂W
  
  例如:
    W₇  = 0.42 - 0.0003 × (+0.03)  = 0.41     ← 调小
    W₂₃ = -0.15 - 0.0003 × (-0.15) = -0.105   ← 调大

重复几万次:
  下次见类似数据 → 预测 3.8 (更接近 4.2)
  再下次 → 预测 4.0
  万次后 → 预测 4.18 (很准了)
```

### 4.3 模型学到了什么？

假设真实规律：**模头 T₃₇ 比 T₃₆ 高 > 5°C → 边压下降 0.3**

```
训练数据中有这个模式:
  样本1: T₃₆=195, T₃₇=201, 边压=3.8  ← 温差 6°C，偏低
  样本2: T₃₆=198, T₃₇=203, 边压=3.7  ← 温差 5°C，偏低
  样本3: T₃₆=197, T₃₇=199, 边压=4.2  ← 温差 2°C，正常

模型自动发现: 把 T₃₇ 的权重调大 → 这些样本的误差减小
不需要人告诉它规则，自己从数据中发现。
```

## 5. 新场景分析方法论

### 5.1 六步分析框架

```
┌──────────────────────────────────────────────────────────┐
│  步骤1: 判断任务类型                                     │
│    ├── 离散输出 → 分类/语言模型                         │
│    │   └── 损失: CrossEntropy                            │
│    └── 连续输出 → 回归/时序预测                          │
│        ├── 一个数 → MSE/MAE                              │
│        └── 一串数 → MSE(每步)                             │
│                                                          │
│  步骤2: 确定输入输出形状                                  │
│    ├── 只看当前 → seq_len=1 → MLP                        │
│    ├── 短期记忆 → seq_len=10~100 → 序列模型              │
│    └── 超长依赖 → seq_len=1000+ → 需要特殊架构           │
│                                                          │
│  步骤3: 选择架构                                         │
│    ├── 数据<1万, 参数受限 → RWKV/GSA (线性复杂度)        │
│    ├── 数据>10万, 有GPU → Transformer                    │
│    └── 需<1ms推理 → LSTM/GRU                             │
│                                                          │
│  步骤4: 数据预处理                                       │
│    ├── 连续数值 → Z-score 归一化                         │
│    ├── 离散类别 → One-hot 编码                           │
│    ├── 缺失值 → 前向填充 / 丢弃                          │
│    └── ⚠️ 时序数据严禁随机打乱！                         │
│                                                          │
│  步骤5: 选择评估指标                                      │
│    ├── 不容大偏差 → MSE (惩罚二次方)                     │
│    ├── 可容忍偶尔大偏差 → MAE                            │
│    └── 不同量级比较 → MAPE (百分比)                      │
│                                                          │
│  步骤6: 部署考虑                                         │
│    ├── 离线批处理 → 模型可大                            │
│    ├── 实时毫秒级 → ONNX + 轻量模型                      │
│    └── 需适应新工况 → 在线学习 / 定期重训                │
└──────────────────────────────────────────────────────────┘
```

### 5.2 除了损失函数和归一化，还要懂什么？

```
1. 过拟合 (Overfitting) — 最常见陷阱
   ├── 表现: 训练 loss=0.001, 测试 loss=0.5
   ├── 原因: 记住了噪音，没学会规律
   └── 解决: 更多数据 / Dropout / Weight Decay / 早停

2. 数据划分 (Train/Val/Test)
   ├── 训练集 70%: 调整权重
   ├── 验证集 15%: 训练中测试，选超参
   └── 测试集 15%: 最后测一次，绝不能在训练中看

3. 数据泄漏 — 工业场景头号杀手
   ├── ❌ 用测试集的 mean/std 归一化训练集
   ├── ❌ 同一时间段的数据分布在训练和测试集
   └── ❌ 未来信息泄露到过去

4. 特征工程
   ├── 温差 = T₃₇ - T₃₆ (加热均匀度)
   ├── 温度方差 (波动程度)
   ├── 温度 - 熔点 (塑化程度)
   └── 5分钟温度变化率 (趋势)

5. 超参数调优
   ├── 学习率: 最重要，3e-4 是合理起点
   ├── batch_size: 4(Mac 16GB) ~ 64(GPU)
   ├── warmup_steps: 学习率从0慢慢涨
   └── grad_clip: 1.0 是标准值
```

## 6. Thinking 模式的本质

### 6.1 LLM Thinking 怎么做

```
普通模式: 输入 → [模型] → 直接输出答案

Thinking: 输入 → [模型] → 先输出思考 → 再输出答案

本质上，思考 token 和普通 token 是完全一样的东西。
区别全在训练方式。
```

### 6.2 训练方法对比

| | 普通模型 | Thinking 模型 |
|---|---|---|
| **架构** | Transformer/RWKV | **完全相同** |
| **参数量** | 一样 | **完全相同** |
| **训练数据** | 问题→答案 | 问题→答案（允许中间思考） |
| **训练方法** | 监督学习 | 监督 + **强化学习 (GRPO)** |
| **推理** | 直接输出 | 生成思考 → 生成答案 |

### 6.3 强化学习怎么做

```python
# 伪代码
for epoch in range(N):
    thoughts, answer = model.generate(question)
    if answer == correct:
        reward = +1.0   # 奖励这种思考方式
    else:
        reward = -0.5   # 惩罚这种思考方式
    model.update(reward)  # 让模型更倾向被奖励的思考方式
```

### 6.4 迁移到工业预测的三种方案

| 方案 | 做法 | 复杂度 | 效果 |
|---|---|---|---|
| **A. 多步预测** | 先预测中间物理状态 → 再预测边压 | ⭐⭐⭐ 高 | 捕捉因果链 |
| **B. 多步回滚** | 预测下一步传感器 → 基于预测做判断 | ⭐⭐ 中 | 自监督 |
| **C. 多头输出** | 同时预测边压 + 温度 + 压力 + 稳定性 | ⭐ 低 | **性价比最高** |

## 7. 方案 C — 多任务学习

### 7.1 核心原理

```
单任务: 66个传感器 → [黑盒] → 边压值
  问题: 模型只为"猜对答案"而学习，可能学的是表面相关

多任务: 66个传感器 → [共享理解层] → ┬ 边压（主任务）
                                       ├ 温度走向（辅助1）
                                       ├ 压力变化（辅助2）
                                       └ 稳定性  （辅助3）
  原理: 为了同时做对四件事，共享层必须学到更深层规律
```

**类比**: 学开车时同时要求"到目的地 + 车速稳 + 保持车距 + 不压线"  
→ 被迫学会通用驾驶能力，而非死记一条路。

### 7.2 数学原理

```
Loss = MSE(边压) + 0.2×MSE(温度) + 0.15×MSE(压力) + 0.1×MSE(稳定性)

共享层必须在同一个 384 维向量里同时编码:
  - "什么影响边压"
  - "什么推动温度变化"
  - "什么导致压力波动"
  
三种信息的"交集"往往就是真正的因果规律。
```

### 7.3 完整模型定义

```python
class IndustrialGSAModel(nn.Module):
    def __init__(self, cfg):
        super().__init__()
        # 输入投影: 66 → 384
        self.input_proj = nn.Linear(cfg.n_features, cfg.n_embd)
        
        # GSA 主体 (不动)
        self.ln0 = nn.LayerNorm(cfg.n_embd)
        self.blocks = nn.ModuleList([
            GSA_Block(cfg, i) for i in range(cfg.n_layer)
        ])
        self.ln_out = nn.LayerNorm(cfg.n_embd)
        
        # 四个输出头
        self.head_edge_pressure = nn.Linear(cfg.n_embd, 1)   # 主: 边压
        self.head_temp_trend    = nn.Linear(cfg.n_embd, 50)  # 辅1: 温度
        self.head_pressure      = nn.Linear(cfg.n_embd, 10)  # 辅2: 压力
        self.head_stability     = nn.Linear(cfg.n_embd, 1)   # 辅3: 稳定性
    
    def forward(self, x):  # x: (B, T, 66)
        x = self.ln0(self.input_proj(x))         # 投影
        for block in self.blocks:
            x = block(x)                          # 序列建模
        x_last = self.ln_out(x[:, -1, :])        # 取最后一步
        
        return {
            "edge_pressure": self.head_edge_pressure(x_last).squeeze(-1),
            "temp_trend":    self.head_temp_trend(x_last),
            "pressure":      self.head_pressure(x_last),
            "stability":     torch.sigmoid(self.head_stability(x_last)).squeeze(-1),
        }
```

### 7.4 模型"思考"过程推演

```
输入: 20 步传感器读数 (发现 T₃₇ 异常偏高 215°C vs 正常 205°C)

GSA Block 0: 门控打分
  T₃₇ 分数: 0.93  ← 模型注意到异常
  P₇  分数: 0.78  ← 模型注意到相关压力异常
  其余 48 个温度点: 0.1~0.3

GSA Block 1~7: 层层抽象
  第1-2层: 识别局部异常
  第3-4层: 建立跨传感器关联 (T₃₇↑ → P₇↑)
  第5-6层: 形成物理直觉 (高温 → 熔体变稀 → 影响成型)
  第7-8层: 生成预测基础

输出:
  边压:  3.75 (低于正常 3.8~4.2) ⚠️
  温度:  T₃₇ 预计继续升至 218°C
  稳定性: 0.31 (偏低，需关注)
```

## 8. 物理公式混合

### 8.1 五种方案

```
┌──────────────┬──────────────────┬──────────┬──────────┬──────────┐
│ 方案         │ 公式参与方式     │ 训练改动 │ 推理改动 │ 效果     │
├──────────────┼──────────────────┼──────────┼──────────┼──────────┤
│ A. 特征增强  │ 公式结果当输入   │ 极小     │ 极小     │ ⭐⭐     │
│ B. 残差学习  │ 模型学"误差"    │ 小       │ 小       │ ⭐⭐⭐   │  ← 推荐
│ C. 物理约束  │ 公式惩罚模型     │ 中       │ 无       │ ⭐⭐⭐   │
│ D. 可微分层  │ 公式嵌入网络     │ 大       │ 大       │ ⭐⭐⭐⭐  │
│ E. 混合输出  │ 部分直接用公式   │ 中       │ 中       │ ⭐⭐⭐   │
└──────────────┴──────────────────┴──────────┴──────────┴──────────┘
```

### 8.2 方案 A：特征增强

把物理公式的输出当成"额外传感器"喂给模型。

```python
physics_features = [
    过热度 = 温度 - 材料熔点,        # 50个
    熔体粘度 = Arrhenius(温度, 熔点), # 1个
    体积流量 = 流速 × 截面积,         # 1个
    最大温差,                          # 1个
    压力波动,                          # 1个
    最接近熔点的温度距离,              # 1个
]
# 输入从 66 维 → 72 维
```

### 8.3 方案 B：残差学习（推荐）

**模型只学"物理公式解释不了的部分"**。

```python
# 训练
physics_estimate = physics_formula(sensors)      # 公式算出一个基准值
residual_true = true_edge_pressure - physics_estimate  # 真实误差
model learns to predict residual_true            # 模型学误差

# 推理
final_prediction = physics_formula(sensors) + model_residual_prediction
```

**好处**：
- 模型失效时，物理公式还能给合理值
- 可解释性强：输出显示"公式基准 3.90 + 模型修正 -0.15 = 3.75"
- 模型工作量大幅减少

### 8.4 方案 C：物理约束损失

不改变模型结构，在损失函数里加惩罚项：

```python
loss = loss_main                          # 预测误差
     + 0.1 × loss_flow_pressure           # 流速↑ → 边压应该↑ (正相关约束)
     + 0.05 × loss_melting                # 熔得好 → 边压应该稳定
```

### 8.5 推荐路线

```
第1阶段 (1-2天): A + B 组合
  └── 改动最小，效果立竿见影

第2阶段 (1周): 加 C (物理约束)
  └── 进一步提升稳健性

第3阶段 (长期): 探索 D (可微分物理层)
  └── 自动从数据校准物理参数
```

### 9.1 数据预处理

```python
# 滑动窗口: 连续序列切成训练样本
for i in range(len(data) - seq_len - 1):
    X.append(data[i : i+20, :66])         # 过去 20 步传感器
    y_main.append(data[i+19, -1])          # 当前边压
    y_temp.append(data[i+20, :50])         # 下一步温度 (辅助)
    y_press.append(data[i+20, 50:60])      # 下一步压力 (辅助)
    y_stability.append(1/(1+std(边压历史)))  # 稳定性 (辅助)

# 归一化: 每个特征独立 Z-score
normalizer.fit(X_train)  # 只用训练集算 mean/std
X_norm = (X - mean) / std
```

### 9.2 训练流程

```python
for epoch in range(epochs):
    for batch in loader:
        # 1. 物理公式部分
        physics_base = physics_formula(raw_sensors[:, -1, :])
        
        # 2. 物理特征增强
        physics_feat = compute_physics_features(raw_sensors)
        combined = cat([norm(raw_sensors), physics_feat], dim=-1)
        
        # 3. 模型前向
        outputs = model(combined)
        
        # 4. 最终预测 = 物理基准 + 模型残差
        edge_pred = physics_base + outputs["residual"]
        
        # 5. 多任务损失
        loss = MSE(edge_pred, y_true)              # 主任务
             + 0.2 * MSE(temp_pred, y_temp)        # 辅1: 温度
             + 0.15 * MSE(press_pred, y_press)     # 辅2: 压力
             + 0.1 * MSE(stab_pred, y_stability)   # 辅3: 稳定性
             + physics_penalty(outputs, sensors)    # 物理约束
        
        # 6. 反向传播
        loss.backward()
        optimizer.step()
```

## 附录 A：关键术语速查

| 术语 | 英文 | 一句话解释 |
|---|---|---|
| 分词器 | Tokenizer | 文字 ↔ 数字的翻译器 |
| BPE | Byte Pair Encoding | 按频率合并字符对的智能分词 |
| 嵌入层 | Embedding | 把 token ID 变成稠密向量 |
| 注意力 | Attention | 让模型"关注"重要的输入部分 |
| 门控 | Gate | sigmoid 输出 0~1，控制信息流 |
| 残差连接 | Residual | 输出 = 输入 + 变换，防梯度消失 |
| 前向传播 | Forward | 输入 → 层层计算 → 输出 |
| 反向传播 | Backward | 误差沿网络传回，算每个权重的贡献 |
| 梯度下降 | Gradient Descent | 沿"误差减小最快方向"微调权重 |
| 学习率 | Learning Rate | 每步调多大的"步长" |
| 归一化 | Normalization | 把所有数值缩放到相近范围 |
| 损失函数 | Loss Function | 衡量预测差多少 |
| 过拟合 | Overfitting | 记住了噪音，没学会规律 |
| 欠拟合 | Underfitting | 太简单，规律都没学够 |
| 回归 | Regression | 预测连续数值（如边压=3.75） |
| 分类 | Classification | 预测离散类别（如下一个token） |
| 强化学习 | RL | 给奖励/惩罚，让模型自己探索 |
| 多任务学习 | Multi-Task | 同时学多个目标，互相促进 |
| 物理信息 | Physics-Informed | 把已知公式融入机器学习 |
